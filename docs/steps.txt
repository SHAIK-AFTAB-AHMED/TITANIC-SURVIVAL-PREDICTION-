TITANIC SURVIVAL PREDICTION WORKFLOW :

    PHASE 1: PROBLEM UNDERSTANDING
        Step 1: 
            Define the Problem
            Identify the task as a Supervised Learning – Binary Classification problem
            Goal: 
                Predict whether a passenger survived or not
            Output classes:
                0 → Not Survived
                1 → Survived

    PHASE 2: DATA ACQUISITION & EXPLORATION
        Step 2: 
            Load Titanic dataset into a DataFrame
            Verify:
            Number of rows (samples)
            Number of columns (features)
        Step 3: 
            Understand the Dataset Structure
            Identify:
                Numerical features
                Categorical features
                Target variable (Survived)
            Inspect:
                Column names
                Data types
                Sample records
        Step 4: 
            Exploratory Data Analysis (EDA)
            Analyze:
                Survival distribution
                Feature-wise survival impact (Sex, Pclass, Age, Fare, etc.)
            Detect:
                Outliers
                Skewed distributions
                Irrelevant columns

    PHASE 3: DATA PREPROCESSING
        Step 5: 
            Handle Missing Values
            Identify missing values in each column
            Apply appropriate strategy:
            Numerical → Mean / Median
            Categorical → Mode
        Step 6: 
            Remove Irrelevant or Redundant Features
            Drop columns that:
            Do not contribute to prediction
            Are identifiers (e.g., PassengerId, Name, Ticket)
            Keep only meaningful predictive attributes
        Step 7:
            Encode Categorical Variables
            Convert categorical data into numerical form:
                Sex → Binary encoding
                Embarked → Label / One-Hot encoding
            Ensure model can process the data numerically
        Step 8:
            Feature Scaling 
            Normalize or standardize numerical features:
                Age
                Fare
            Important for distance-based models

    PHASE 4: FEATURE SELECTION & DATA PREPARATION
        Step 9: Feature Selection
            Select most relevant features using:
            Correlation analysis
            Domain knowledge
            Remove weak or noisy features

        Step 10: Split Dataset
            Divide dataset into:
                Training set
                Testing set
            Typical ratio:
            70:30 or 80:20
            Prevent data leakage

    PHASE 5: MODEL BUILDING
        Step 11: Choose Classification Algorithm
            Select suitable model:
                Logistic Regression
                Decision Tree
                Naive Bayes
                KNN
            Justify choice based on:
                Dataset size
                Feature type
                Interpretability
        Step 12: Train the Model
            Fit the selected algorithm using training data
            Learn patterns between features and survival outcome

    PHASE 6: MODEL EVALUATION
        Step 13: Make Predictions
            Predict survival on test dataset
        Step 14: Evaluate Model Performance
            Use evaluation metrics:
            Accuracy (Explain what accuracy means in this context)
            Confusion Matrix
            Precision
            Recall
            F1-Score

        Step 15: Model Optimization 
            Improve performance using:
                Hyperparameter tuning
                Feature refinement
                Algorithm comparison

    PHASE 7: FINAL MODEL USAGE
        Step 16: Test Model with Custom Input
            Accept new passenger data as input
            Preprocess input same as training data
            Predict survival result

        Step 17: Final Conclusion
            Summarize:
            Model performance
            Key features affecting survival
            Limitations
            Mention possible future improvement
